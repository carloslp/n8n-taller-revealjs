Introducción a n8n

Automatización de Flujos de Trabajo para Equipos de Desarrollo

¿Qué es n8n?

n8n (se pronuncia "en-ocho-en") te ayuda a conectar cualquier aplicación con API con otra, y manipular sus datos con poco o nada de código.

  • Personalizable: Workflows altamente flexibles y opción de crear nodos personalizados.
  • Conveniente: Prueba n8n usando npm, Docker o la opción Cloud si prefieres no gestionar la infraestructura.
  • Enfocado en privacidad: Aloja n8n tú mismo para máxima privacidad y seguridad.

Conceptos Clave

Nodos (Nodes)

Bloques de construcción. Un nodo puede ser un trigger, una app (GitHub, AWS S3) o una función lógica (IF, Merge, Code).

Items (Estructura de Datos)

La data fluye entre nodos como un array de objetos JSON. Cada nodo procesa estos items y pasa el resultado al siguiente.

Workflows

El lienzo donde se conectan los nodos. (Dato curioso: los workflows son JSON, ¡perfectos para versionar con Git!)

Nodos Esenciales

Disparadores (Triggers)

  • Cron: Ejecuta workflows en un horario fijo.
  • Webhook: Inicia un workflow con una petición HTTP.
  • App Triggers: (GitHub, GitLab, Stripe, etc.)

Lógica Principal

  • HTTP Request: La navaja suiza para cualquier API.
  • Code: Ejecuta tu propio código JS/TS.
  • IF / Switch: Bifurca el flujo basado en condiciones.
  • Merge: Combina datos de diferentes ramas.

Nodos Populares

Integraciones Comunes

  • Bases de Datos: Postgres, MySQL, MongoDB...
  • Mensajería: Slack, Discord, Telegram, Email.
  • Cloud: AWS (S3, Lambda), GCP, Azure.

Procesamiento de Datos

  • Set: Modifica o crea campos en los items.
  • Edit Fields: Renombra, elimina o reordena campos.
  • Read Binary File: Lee archivos para procesarlos.

Un Playground para Desarrolladores

n8n no es una caja cerrada, es un framework.

Nodo `Code`

Ejecuta JS/Node.js arbitrario. Puedes usar `require()` para importar librerías de npm. Ideal para transformaciones de datos complejas o lógica de negocio específica.

Nodo `HTTP Request`

El conector universal. Interactúa con cualquier API REST, interna o externa. Soporta OAuth2, Bearer Token, etc.

Nodos Personalizados

¿Necesitas una integración reutilizable? Construye tu propio nodo con el Node Development Kit (NDK). Es un simple archivo `*.node.ts`.

API de n8n

Gestiona workflows, credenciales y ejecuciones de forma programática a través de la propia API REST de n8n.

Arquitectura y Auto-alojamiento

Control total sobre la ejecución y los datos.

Opciones de Despliegue: Docker, Docker Compose, Kubernetes (Helm Chart), o directamente con npm.

Escalabilidad: n8n puede escalar horizontalmente configurando múltiples "workers" para procesar ejecuciones en paralelo.

Beneficios para Devs:

  • Los datos y credenciales nunca salen de tu VPC/red.
  • Conecta de forma segura con servicios internos (DBs, APIs privadas).
  • Sin límites artificiales de ejecución. El límite lo pone tu infraestructura.

¿Por qué n8n para un equipo de Devs?

Source-available y Extensible

El código está disponible. Puedes entender cómo funciona, depurar y extenderlo. No hay cajas negras.

Integración con el Ecosistema Dev

Los workflows como código (JSON/Git), despliegue con Docker/K8s, y la capacidad de construir tus propias herramientas.

Seguridad y Cumplimiento

Al ser auto-alojado, cumple con las políticas de seguridad más estrictas. Ideal para entornos regulados.

Acelera el "Glue Code"

Reduce drásticamente el tiempo dedicado a escribir código repetitivo para conectar servicios. Permite enfocarse en la lógica de negocio.

Primeros Pasos

¡Empieza a automatizar en minutos!

  • 1. Levanta una instancia local con Docker Compose: 
    
	services:
		n8n:
			container_name: n8n
			image: docker.n8n.io/n8nio/n8n
			ports:
				- "5678:5678"
			environment:
				- GENERIC_TIMEZONE=America/Mazatlan
				- TZ=America/Mazatlan
				- N8N_SECURE_COOKIE=false
				- NODE_ENV=production
				- N8N_PROTOCOL=https
				- N8N_HOST=
				- N8N_DIAGNOSTICS_ENABLED=false
				- N8N_EMAIL_MODE=smtp
				- N8N_SMTP_HOST=
				- N8N_SMTP_PORT=
				- N8N_SMTP_USER=
				- N8N_SMTP_PASS=
				- N8N_SMTP_SENDER
				- N8N_SMTP_SSL=false
				- WEBHOOK_URL=
				- N8N_EDITOR_BASE_URL=
			volumes:
				- ./data:/home/node/.n8n
  • 2. Explora los workflows de la comunidad en n8n.io/workflows
  • 3. Revisa el código y contribuye en GitHub.
